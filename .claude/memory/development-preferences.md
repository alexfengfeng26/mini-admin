# 开发偏好设置

## 热重载开发模式

**重要：开发时不要随意重启程序**

- 后端使用 `ts-node-dev` 支持热重载，修改代码后自动重启
- 前端使用 Vite HMR，修改组件后热更新
- 只有在必要时才重启服务器（如修改环境变量、数据库迁移等）
- 优先使用热重载/热更新来保持开发状态的连续性

## 禁止的操作

以下操作会破坏热重载，**未经明确同意不要执行**：
- 杀死开发服务器进程
- 执行 `npm run dev` 重启命令
- 关闭并重新启动终端会话

## 安全操作

以下操作是安全的，不会破坏热重载：
- 修改源代码文件（TypeScript/Vue 文件）
- 修改配置文件（部分配置可能需要重启，会提前询问）
- 执行构建命令（`npm run build`）用于测试
- 数据库相关操作（`prisma migrate`、`prisma studio`）
